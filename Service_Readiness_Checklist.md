# Service Readiness Checklist

## 1) 핵심 사용자 시나리오 (필수)
- [ ] 홈에서 해커톤 탐색 CTA로 이동된다.
- [ ] 해커톤 목록에서 필터/정렬/검색 조합이 의도대로 동작한다.
- [ ] 해커톤 상세에서 팀 구성 -> 제출 -> 리더보드 반영 흐름이 끊기지 않는다.
- [ ] 캠프에서 팀 생성/수정/삭제 권한이 올바르게 동작한다.
- [ ] 랭킹에서 보드/기간/정렬 필터가 정확히 반영된다.

## 2) 링크 공유/재진입 시나리오
- [ ] 해커톤 목록의 "현재 뷰 링크 복사" URL이 같은 결과를 재현한다.
- [ ] 캠프의 "현재 뷰 링크 복사" URL이 같은 결과를 재현한다.
- [ ] 랭킹의 "현재 뷰 링크 복사" URL이 같은 결과를 재현한다.

## 3) 접근성
- [ ] 키보드만으로 상단 메뉴 -> 본문 -> 주요 CTA 이동이 가능하다.
- [ ] Skip link("본문으로 바로 이동")이 동작한다.
- [ ] 포커스 링이 버튼/링크/입력에서 충분히 보인다.
- [ ] 로딩/에러/빈상태에서 상태 메시지가 스크린리더 친화적으로 노출된다.

## 4) 반응형
- [ ] 375px / 390px / 430px에서 주요 카드/필터가 깨지지 않는다.
- [ ] 768px에서 사이드바 및 필터 그리드가 자연스럽게 재배치된다.
- [ ] 1024px 이상에서 헤더/콘텐츠 폭이 과도하게 늘어나지 않는다.

## 5) 데이터/상태
- [ ] 새 제출이 submissions와 leaderboard 모두에 반영된다.
- [ ] 팀 합류 승인 시 팀 인원 수가 정확히 증가한다.
- [ ] localStorage에 저장된 메시지/신청/요청 데이터가 새로고침 후 유지된다.

## 6) 품질 게이트
- [ ] `npm run build` 성공
- [ ] 콘솔 에러 0건
- [ ] 치명적 UX 버그 0건(제출 불가, 이동 불가, 데이터 소실)

## 7) 배포 전 마지막 점검
- [ ] Vercel preview URL에서 위 체크리스트를 1회 재실행
- [ ] 팀 역할별(탐색/팀관리/제출) 담당자가 각자 1회 검수
- [ ] 최종 제출용 스크린샷 5장 이상 확보 (홈/목록/상세/캠프/랭킹)
